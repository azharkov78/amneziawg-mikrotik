# Changelog

## v1.0.4 (2026-03-06)

### Исправления
- Исправлена ошибка "address already in use" после нескольких часов работы
  - **Ложные таймауты**: `lastActive` обновлялся раз в 256 пакетов в batch-режиме, что вызывало ложный таймаут через ~3 мин при работающем трафике (keepalive раз в 25с × 256 = 107 мин между обновлениями, а таймаут — 180с). Теперь `lastActive` ставится на каждый batch
  - **SO_REUSEADDR**: добавлена установка `SO_REUSEADDR` при bind src-порта — страховка от `EADDRINUSE` при реконнекте, пока dup'd fd ещё не закрыт
  - **Порядок close/reconnect**: в s2c batch `recvFD` теперь закрывается ДО вызова `reconnectRemote`, освобождая порт для нового `bind`. Ранее dup'd fd держал порт до завершения reconnect, вызывая `EADDRINUSE`

## v1.0.3 (2026-03-03)

### Исправления
- Исправлена потеря соединения при активном использовании и wireguard rekey в это время 
  - Причина: Junk-пакеты создавали burst при каждом rekey, вызывая потерю handshake init на нестабильных каналах
  - Результат: периодические отвалы VPN (~каждые 6 мин)
  - Теперь junk и CPS отправляются только при первом хэндшейке (cold start / reconnect)
  - При rekey отправляется только трансформированный handshake init

## v1.0.1 (2026-03-02)

### Производительность
- Пакетная отправка/приём через recvmmsg/sendmmsg (до 128 пакетов за один системный вызов)
- UDP GRO/GSO: приём и отправка нескольких пакетов одним вызовом ядра (с автофоллбеком)
- Быстрая обработка транспортных пакетов без лишних копирований
- Быстрый генератор случайных чисел (xorshift64) для padding и подмены типов
  - Кольцевой буфер H4 на 256 значений — одно чтение из массива на пакет
  - Циклический буфер 64KB для S4 padding
- c2s zero-copy: sendState без буферов, iovecs → recvBS.bufs, экономия ~224KB RAM
- initRecvFirstAddr в c2s: адрес клиента берётся только из первого пакета батча
- Амортизация lastActive.Store: atomic store раз в 256 пакетов
- Убран SO_BUSY_POLL — мёртвый код на blocking FD
- Размер батча увеличен с 32 до 128 пакетов
- GOMAXPROCS по умолчанию 3 (было 2)
- Отключение сборщика мусора Go по умолчанию (AWG_GOGC для override)

### Функциональность
- AWG_SRC_PORT: привязка source-порта для NAT hole punching
  - Не задан / 0 → auto-режим: порт берётся из первого пакета клиента
  - Число 1-65535 → статический порт

### Исправления
- Надёжное завершение при остановке контейнера
- Уменьшен размер логов (один вызов записи вместо трёх)
